Er is een verscheidenheid aan manieren om bij te dragen aan de ontwikkeling van Semantic. We zijn een erg nieuw project en zijn op zoek naar een enthousiaste en gelijkgestemde groep kern bijdragers.

Publiciteit

Een van de eenvoudigste manieren om Semantic UI te steunen is door het bekender te helpen maken




Bugs Pletten

Voer a.u.b. alle bugs die je tegenkomt wanneer je de bibliotheek gebruikt op in onze Github Issues Tracker.

Zorg ervoor dat je een set aan herhaalbare stappen toevoegt die nodig zijn om het probleem te reproduceren en alle gerelateerde informatie zoals, browser, OS etc. Indien we het probleem niet kunnen reproduceren wordt het erg moeilijk om het probleem op te lossen.

Wanneer je een bug rapportage indient, Fork deze JSFiddle a.u.b. om een test-case te maken. Het heeft Semantic en normalize.css reeds automatisch voor je ingevoegd.

Ideeën

Semantic zoekt naar mensen die kunnen helpen om bij te dragen met nieuwe kern UI componeten en nieuwe extenties kunnen suggereren voor de bibliotheek.

Indien je suggesties hebt voor missende componenten in Semantic die je in toekomstige versies zou willen zien, voeg ze a.u.b. toe aan ons publieke UI Componenten bord. De huidige lijst met aankomende componenten, en hun huidige ontwikkelings status kan op het bijdragers UI bord bekeken worden.

Code

We zijn op zoek naar een aantal goede programmeurs om ons te helpen met het maken van belangrijke nieuwe functies voor een release versie van Semantic. Hier is een overzich van huidige functies in ontwikkeling

CSS Variabelen

We werken aan het porten van Semantic om CSS variabelen te kunnen gebruiken zodat het makkelijker is om the themen. We zijn momenteel van plan om een CSS Pre-processor te gebruiken om bruikbare waardes om te zetten in variabelen. Het huidige plan is om niet mix-ins of pre-processor specifieke code te gebruiken naast variabelen, en om vendor prefixing toe te voegen met behulp van Grunt.

Voorbeeld Layouts

We zijn ernaar aan het kijken om voorbeeld layouts te maken met Semantic om mensen te helpen hun sites sneller te maken, en om de ingewikkeldheden van het gebruik van Semantic UI te leren. Het plan is om uiteindelijk een zuster site learnsemantic.com te maken, welke nuttige functies voor het leren van Semantic zal hebben.

Build Tools

We moeten een customizer voor Semantic maken zodat gebruikers kunnen kiezen welke componenten ze willen gebruiken en custom zips met hun minified, aaneengeschakelde code kunnen maken.

Waarschijnlijk betekend dit het maken van een standalone server die API requests accepteerd voor custom builds van Semantic en een zip retourneert met minified en ongecomprimeerde bron code, of met wat vindingrijkheid een in-page javascript die met behulp van JSZIP hetzelfde doet.

Test Coverage

Semantic gebruikt Jasmine met Karma Test Runner en Travis CI om unit testen te schrijven voor onze UI modules.

We kunnen je hulp gebruiken om onze code coverage te verbeteren door Jasmine tests te schrijven voor modules en functie specificaties te schrijven die kunnen worden gebruikt om tests te schrijven.
