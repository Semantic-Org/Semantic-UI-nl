Wat is Anders

Semantic onderscheiden van de rest

Filosofie

Bouw Responsive Layouts Gemakkelijker
Compleet Ontworpen met EM

Elk component is gedefinieerd met behulp van em en rem zodat componenten simpelweg gaandeweg in grootte aangepast kunnen worden. Wil je dat een menu kleiner wordt op een mobiel? Laat simpelweg z'n font grootte veranderen met behulp van een media query.

Spreekt Voor Zich
Beschrijvend, niet Voorschrijvend

Front end code schrijven zou niet het leren van de naamgevings- of programmeerconventies van een bepaalde ontwikkelaar moeten inhouden.

In plaats van afkortingen, of codificatie van naamgevingsconventies, gebruikt Semantic simpele, algemene taal voor delen van interface elementen, en vertrouwde patronen welke in natuurlijke talen gevonden worden voor het beschrijven van elementen.

Tag ambivalent
Gebruik elke willekeurige html tags die je wilt.

Interface definities in Semantic zijn tag ambivalent. Dat betekend dat je div, article, section, span kan gebruiken zonder dat het de weergave van het element beinvloed. Speciale tags zoals a, table, td behouden echter hun speciale betekenissen in bepaalde omstandigheden.

Krachtige gereedschappen voor het uitdrukken van groepen en collecties.
Herhaal jezelf niet

In het Engels is het veel makkelijker om te zeggen "There are three tall men" (Er zijn drie lange mannen) dan "There is a tall man, a tall man and a tall man" (Er is een lange man, een lange man en een lange man).

In Semantic kunnen element definities uitgedrukt worden in groepen welke gedeelde attributen zoals grootte, kleur en type kunnen hebben, daarmee repetitieve declaraties vermijdend.

Portable en self-contained.
Semantic gebruiken betekent niet een heel framework adopteren, of je codebasis herschrijven

Semantic componenten zijn geheel in een eenduidige stijl geschreven, maar zijn niet onderdeel van een gemandateerde overkoepelende bibliotheek. Vindt je alleen enkele componenten leuk? Geen probleem, gebruik alleen wat je nodig hebt.

UI componenten in Semantic definieren ook optionele en verplichtte koppelingen met andere componenten waar hun gebruik elkaar snijdt. Dat betekent bijvoorbeeld, een popup kan controleren of het CSS animatie component bestaat alvorens de fallback javascript animaties te gebruiken.

Gedeelde taal, verschillende implementaties
Herstijl je site zonder het te Herstructureren

Je site z'n inhoud beschrijven met behulp van een gedeelde taal zoals Semantic, stelt andere ontwikkelaars in staat om UI definities maken die een gedeelde woordenschat gebruiken. Dit betekent dat je je website kan herontwerpen zonder he html aan te passen. Verander eenvoudig de look-and-feel van je UI door een andere UI style definitie te gebruiken.

Alleen het belangrijke spul
Niet het aanrecht

In plaats van het geven van elke mogelijk variatie of gedrag dat zou kunnen bestaan, zijn element definities ontworpen om een startpunt te zijn. Geen overmatig grote downloads met optionele functies die je waarschijnlijk nooit gebruikt, alleen het spul om je te helpen starten.
